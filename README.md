# Yandex Music Migrator

Скрипт для переноса музыки между двумя аккаунтами **Яндекс Музыки**.

## Возможности

Проект позволяет автоматически перенести:

- ❤️ Треки из раздела **«Мне нравится»**;
- 🎵 Все пользовательские плейлисты;
- 📝 Названия плейлистов;
- 🔒 Статус приватности каждого плейлиста.

Для обеспечения стабильной работы при переносе плейлистов скрипт автоматически:

- получает актуальную `revision` плейлиста;
- обновляет её после каждого добавленного трека;
- повторно запрашивает `revision` при возникновении ошибок API.

Проект полностью адаптирован для запуска в **Docker**.

# Настройка

Создайте файл `.env` рядом с `docker-compose.yml`.

Пример:

```env
SOURCE_TOKEN=AQAAAAA...
TARGET_TOKEN=AQAAAAA...
```

Где:

- `SOURCE_TOKEN` — токен аккаунта, с которого производится перенос;
- `TARGET_TOKEN` — токен аккаунта, на который производится перенос.

---

# Запуск

В каталоге проекта выполните:

```bash
docker compose up --build
```

При первом запуске Docker:

1. Соберёт образ.
2. Установит зависимости.
3. Запустит скрипт.
4. Выполнит перенос данных.
5. Завершит работу контейнера.

---

# Логи

Во время выполнения в консоль выводится информация о ходе миграции:

- успешная авторизация;
- количество найденных лайков;
- прогресс переноса лайков;
- список найденных плейлистов;
- создание новых плейлистов;
- количество успешно перенесённых треков;
- возможные ошибки API.

Благодаря переменной

```dockerfile
ENV PYTHONUNBUFFERED=1
```

логи отображаются в режиме реального времени.

# Dockerfile

Используется минимальный образ Python:

```dockerfile
FROM python:3.11-slim
```

Во время сборки:

- копируются зависимости;
- устанавливаются необходимые библиотеки;
- копируется основной скрипт;
- задаётся команда запуска.

---

# docker-compose

Используется простая конфигурация:

```yaml
version: "3.8"

services:
  music-migrator:
    build: .
    container_name: yandex_music_migrator
    env_file:
      - .env
    restart: "no"
```

Контейнер запускается один раз, выполняет миграцию и автоматически завершается.

---

# Алгоритм работы

После запуска выполняются следующие шаги:

1. Чтение токенов из переменных окружения.
2. Авторизация в обоих аккаунтах Яндекс Музыки.
3. Перенос раздела **«Мне нравится»** пакетами по 50 треков.
4. Получение списка пользовательских плейлистов.
5. Создание аналогичных плейлистов на целевом аккаунте.
6. Последовательное добавление всех треков.
7. Автоматическое обновление `revision` после каждой операции.
8. Завершение работы.

# Особенности

- Не требует изменения исходного кода для смены аккаунтов.
- Все секреты хранятся в `.env`.
- Полностью совместим с Docker.
- Поддерживает большое количество треков.
- Автоматически делает задержки между запросами для снижения вероятности получения ограничений API.
- Продолжает выполнение даже при ошибках отдельных треков или плейлистов.

---

# Ограничения

Скрипт переносит только пользовательские данные:

- ✅ «Мне нравится»;
- ✅ пользовательские плейлисты.

Не переносятся:

- история прослушивания;
- рекомендации;
- подписки;
- настройки аккаунта;
- скачанная музыка.
